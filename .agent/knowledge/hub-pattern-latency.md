# Hub/Orchestrator 패턴 응답 시간

## 결론
PM Hub 패턴의 느린 응답 시간은 **버그가 아니라 의도된 트레이드오프**. 업계 표준(CrewAI Hierarchical, LangGraph Supervisor) 70%가 이 패턴 사용.

## 왜 느린가
Hub 루프가 순차 실행이기 때문:
1. PM 첫 응답 (~25s) → 위임 대상 결정
2. 위임 에이전트 실행 (~25-42s) → 작업 수행
3. PM 리뷰 (~25s) → 중간 결과 검증
4. 추가 위임 또는 완료 판단
- 전체 사이클: 2분+ (3단계 위임 기준)

## 왜 안전한가
- PM이 중간 결과를 리뷰하고 다음 위임을 판단
- 잘못된 방향 → PM이 중간에 교정 가능
- 최종 품질을 PM이 보장

## 체감 속도 개선 (구조 변경 없이)
- 🧠 리액션으로 진행 상태 표시 (구현 완료)
- 각 에이전트 응답이 Slack에 즉시 포스팅 → 사용자가 중간 결과 확인 가능

## 참고 수치 (2026-03-26 측정)
- PM 1회 호출: ~25s, output ~1000-2000 tokens
- cacheRead=141K tokens → 프롬프트 캐싱 히트 (비용 90% 절감, 속도 영향 미미)
- MCP 서버 spawn 오버헤드 포함
