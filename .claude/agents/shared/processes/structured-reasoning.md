# Structured Reasoning Protocol

> 출처: Devin AI `<think>` scratchpad 패턴

## 적용 대상
모든 에이전트 — 중요 결정 전 의무 적용.

## 규칙

### 언제 구조화 추론을 수행하는가
다음 상황에서는 반드시 결정 전에 추론 과정을 명시적으로 기록한다:

1. **아키텍처 결정**: 기술 선택, 패턴 결정, 구조 변경
2. **크로스도메인 위임**: 다른 에이전트에게 작업을 넘기기 전
3. **Git 전략**: 브랜치, 머지, 리베이스 방향 결정
4. **예상치 못한 어려움**: 계획대로 진행이 안 될 때 방향 전환
5. **완료 선언 직전**: 모든 요구사항이 충족되었는지 최종 점검

### 추론 형식

Slack 보고 또는 메모리 기록 시 다음 형식을 사용한다:

```
**판단 근거:**
- 현재 상태: [관찰한 사실]
- 선택지: [A] vs [B] (vs [C])
- 선택: [결정]
- 이유: [왜 이 선택이 최선인가]
- 리스크: [이 선택의 위험 요소]
```

### 하지 않을 때
- 단순 파일 수정 (타이포, 명백한 버그)
- 이미 계획이 승인된 구현 단계
- 일상적 보고/업데이트
