# Verification Before Completion

> **Iron Law**: 완료 주장 전에 반드시 검증 증거를 확보하라. 예외 없음.

## 원칙

모든 에이전트는 작업 완료를 선언하기 전에 **실제 검증 결과**를 확보해야 한다. 이전 실행 결과, 다른 에이전트의 보고, 또는 가정에 기반한 완료 주장은 금지된다.

## Gate Function: 5단계 검증

완료를 선언하려면 반드시 이 5단계를 순서대로 통과해야 한다:

### 1. IDENTIFY — 검증 대상 식별
- 내가 "완료"라고 주장하려는 것이 정확히 무엇인가?
- 어떤 증거가 있어야 완료를 증명할 수 있는가?

### 2. RUN — 검증 실행
- 검증 명령어/프로세스를 **지금 이 시점에** 직접 실행한다
- 이전 실행 결과를 재사용하지 않는다

### 3. READ — 결과 확인
- 실행 결과를 **끝까지** 읽는다
- 성공 메시지만 보지 않고, 경고와 부분 실패도 확인한다

### 4. VERIFY — 요구사항 대조
- 결과가 원래 요구사항을 **모두** 충족하는지 확인한다
- 부분 충족은 완료가 아니다

### 5. CLAIM — 증거와 함께 완료 선언
- Slack에 완료를 보고할 때, 검증 증거를 함께 첨부한다
- "완료했습니다" + 증거 = 유효한 완료
- "완료했습니다" (증거 없음) = 무효

## 작업 유형별 검증 테이블

| 작업 유형 | 담당 에이전트 | 필수 검증 |
|-----------|-------------|----------|
| 코드 구현 | Frontend, Backend | 테스트 통과 + 빌드 성공 + lint 통과 |
| API 개발 | Backend | 엔드포인트 테스트 + 응답 스키마 검증 + 에러 케이스 확인 |
| 디자인 시안 | Designer | 디자인 스펙 체크리스트 완료 + 접근성 검증 + 반응형 확인 |
| 보안 리뷰 | SecOps | 취약점 스캔 실행 + 발견 사항 재현 + 수정 확인 |
| 시장 조사 | Researcher | 출처 교차 검증 + 데이터 유효성 확인 + 최신성 검증 |
| 기획 문서 | PM | 요구사항 체크리스트 완료 + 이해관계자 리뷰 + 기술 검토 |
| 버그 수정 | Frontend, Backend | 원인 확인 + 수정 테스트 + 회귀 테스트 |
| 인프라 변경 | Backend, SecOps | 배포 성공 + 헬스체크 통과 + 모니터링 정상 |

## Red Flags — 이 신호가 보이면 멈춰라

다음 표현이 검증 **이전에** 나오면 Gate Function 위반이다:

- ❌ "완료했습니다" / "Done!" / "Fixed!"
- ❌ "잘 될 겁니다" / "should work" / "probably works"
- ❌ "문제없을 것 같습니다" / "seems fine"
- ❌ "이전에 확인했으니까..." / "already checked"
- ❌ 만족감 표현 ("Perfect!", "Great!") → 검증 실행 전

## Slack 완료 보고 패턴

### ❌ 잘못된 보고
```
@Marge 작업 완료했습니다. 잘 동작합니다.
```

### ✅ 올바른 보고
```
@Marge 작업 완료했습니다.

검증 결과:
- 테스트: 24/24 통과 (스크린샷 첨부)
- 빌드: 성공 (로그 첨부)
- 요구사항 체크리스트: 5/5 항목 충족
  - [x] 사용자 목록 페이지 렌더링
  - [x] 페이지네이션 동작
  - [x] 검색 필터 동작
  - [x] 반응형 레이아웃
  - [x] 키보드 네비게이션
```

## 에스컬레이션

- 검증 중 예상치 못한 실패 발견 → 즉시 Slack 스레드에 보고하고 수정 진행
- 검증 환경 자체에 문제 → sid에게 에스컬레이션
- 부분적으로만 완료 가능 → "부분 완료"로 보고하고 남은 항목 명시

## 합리화 방어 테이블

이런 생각이 들면 검증을 건너뛰려는 것이다:

| 핑계 | 현실 |
|------|------|
| "이제 작동할 것이다" | 검증 명령 실행해라 |
| "확신한다" | 확신 ≠ 증거 |
| "이번만 건너뛰자" | 예외 없음 |
| "린터/타입체크 통과했다" | 린터 ≠ 런타임 |
| "에이전트가 성공이라 했다" | 독립적으로 검증해라 |
| "이전에 확인했다" | 이전 결과 ≠ 현재 증거 |
| "부분 검사로 충분하다" | 부분은 아무것도 증명 못함 |
| "피곤하니까 / 급하니까" | 피로와 긴급도 ≠ 면제 |

## 이 프로세스는 선택이 아니다

"이번만 건너뛰자"는 없다. 간단한 작업도, 긴급한 작업도, 모든 작업에 이 프로세스를 적용한다. 검증 없는 완료 주장은 완료가 아니다.
