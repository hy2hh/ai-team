# 코드 분석/보고 검증 상세

분석 보고서에 수치, 비교, 최적화 제안이 포함된 경우 아래를 반드시 확인:

## 1. 수치 주장 → 소스 대조

- "인덱스 4개" → 실제 코드에서 `CREATE INDEX`를 전수 카운트
- 반드시 `파일명:라인번호` 인용 (예: "인덱스 6개 — db.ts:52-68")
- 주장한 수치와 코드 상 수치가 다르면 → 수치 수정 후 재보고

## 2. 조건 분기 → 전수 경로 분석

- if/else, 조건부 실행이 있으면 최소/최대 실행 경로를 모두 명시
- "최소 N개" 주장은 모든 조건을 false로 놓았을 때의 경로 기준
- 빠진 경로가 있으면 → 해당 경로 추가 분석 후 재보고

## 3. 최적화 제안 → 원본과 diff 비교

- 원본 쿼리/코드와 제안 코드를 나란히 비교
- WHERE 절, JOIN 조건 등 필터링 로직의 차이를 강조
- 원본에 있는 조건(`AND id != ?` 등)이 제안에서 누락되면 버그 → 제안 수정
