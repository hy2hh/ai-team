# 리뷰어 매트릭스

| 리뷰 대상 | 주 리뷰어 | 보조 리뷰어 | 리뷰 관점 |
|-----------|----------|-----------|----------|
| Frontend 코드 | @Homer | @Krusty | API 계약 준수 / UI 구현 충실도 |
| Backend 코드 | @Wiggum | @Bart | 보안 / API 소비자 관점 |
| API 계약 변경 | @Bart + @Homer | @Wiggum | 양측 호환성 / 보안 |
| 보안 관련 코드 | @Wiggum | @Homer | 취약점 / 아키텍처 |
| UI 구현 | @Krusty | @Bart | 디자인 스펙 일치 / 접근성 |

## 구조적 품질 검증 (Code Quality 리뷰 시 추가 체크)

- [ ] 각 파일이 하나의 명확한 책임을 가지며 잘 정의된 인터페이스를 노출하는가?
- [ ] 단위가 독립적으로 이해·테스트 가능하게 분해되었는가?
- [ ] 구현이 계획서의 파일 구조를 따르는가?
- [ ] 이 변경이 새로 만든 파일이 이미 크거나, 기존 파일을 크게 성장시켰는가?
- [ ] 테스트가 mock이 아닌 실제 동작을 검증하는가? (`.claude/context/qa/testing-anti-patterns-checklist.md` 참조)
