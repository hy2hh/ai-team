# BUG-7 / BUG-5 개입 분석 및 메시지 초안

## 상황 분석

### BUG-7: Homer (Backend) — "수정 방향을 검토했습니다. 이대로 진행할까요?"

**위반 규칙:**
1. **CLAUDE.md > Work Completion Rules > 자율 실행**: "도구로 직접 할 수 있는 작업은 사용자에게 요청하지 말고 직접 수행"
2. **collaboration-rules.md > Auto-Commit Rule**: "'커밋할까요?' 질문 금지. 직접 실행"
3. **Homer 페르소나 > 완료 선언 규칙**: "계획만 세우고 구현 완료라고 보고" 금지 패턴에 해당
4. **Homer 페르소나 > 구현 착수 전 규칙**: "실패 시나리오 정의 → 현재 상태 확인 → 성공 기준 대조" 후 바로 구현 착수해야 함

**진단**: Homer가 "진행할까요?" 안티패턴을 정확히 재현하고 있다. 버그 수정이 위임된 상태에서 검토만 하고 실행하지 않은 채 확인을 요청하는 것은 자율 실행 원칙 위반이다. Homer는 수정 방향이 명확하다면 즉시 구현 → 테스트 → 커밋 → 보고해야 한다.

**개입 수준**: 직접 개입 필요 (명확한 규칙 위반)

---

### BUG-5: Lisa (Researcher) — "리서치 A/B 중 어떤 방향으로 조사할까요?"

**위반 규칙:**
1. **CLAUDE.md > Work Completion Rules > 자율 실행**: 동일 규칙 위반
2. **collaboration-rules.md > Escalation Rules**: "모호한 작업: sid에게 확인" — 리서치 방향이 불명확하면 sid에게 에스컬레이션해야 하지, 위임자에게 역질문하면 안 됨

**진단**: Lisa의 경우 약간 다른 뉘앙스가 있다. 리서치 방향 A/B가 본질적으로 다른 결과를 낳는 경우, 방향 확인은 합리적일 수 있다. 그러나 프레임워크 규칙상 에이전트는 자율적으로 판단하고 실행해야 한다. 방향이 정말 불명확하면 sid에게 에스컬레이션하는 것이 올바른 절차이다.

**개입 수준**: 직접 개입 필요 (자율 실행 원칙 재교육 + 올바른 에스컬레이션 경로 안내)

---

## 개입 메시지 초안

### Homer에게 보낼 메시지

```
@Homer BUG-7 수정이 위임된 상태에서 "진행할까요?"는 자율 실행 원칙 위반입니다.

규칙 확인:
- CLAUDE.md: "도구로 직접 할 수 있는 작업은 사용자에게 요청하지 말고 직접 수행"
- collaboration-rules.md: "'커밋할까요?' 질문 금지. 직접 실행"
- homer.md: 계획만 세우고 완료 보고하는 패턴 금지

수정 방향이 검토 완료되었다면 즉시 실행하세요:
1. 코드 수정
2. 서버 실행하여 동작 확인
3. curl로 엔드포인트 응답 검증
4. 커밋 + push
5. 완료 보고 (homer.md 완료 보고 형식 준수)

확인 질문 없이 직접 진행해주세요.
```

### Lisa에게 보낼 메시지

```
@Lisa BUG-5 리서치가 위임된 상태에서 방향 선택을 역질문하는 것은 자율 실행 원칙 위반입니다.

규칙 확인:
- CLAUDE.md: "도구로 직접 할 수 있는 작업은 사용자에게 요청하지 말고 직접 수행"
- collaboration-rules.md 에스컬레이션 규칙: 모호한 작업은 sid에게 확인

올바른 대응:
1. 두 방향 모두 조사하여 비교 분석 제공 (리서처의 핵심 가치)
2. 또는 전문적 판단으로 더 적합한 방향을 선택하고 근거와 함께 결과 보고
3. 정말 판단 불가능한 경우 → sid에게 에스컬레이션 (위임자에게 역질문 아님)

리서처로서 방향 A/B의 장단점을 분석하는 것 자체가 리서치 결과물입니다. 양쪽 모두 간략히 조사한 후 추천 방향과 근거를 포함하여 보고해주세요.
```

## 종합 판단

| 항목 | Homer (BUG-7) | Lisa (BUG-5) |
|------|--------------|--------------|
| 위반 유형 | "진행할까요?" 안티패턴 | 리서치 방향 역질문 |
| 심각도 | 높음 (명시적 금지 패턴) | 중간 (자율 실행 위반) |
| 근본 원인 | 자율 실행 원칙 미준수 | 자율 판단 회피 |
| 개입 방식 | 규칙 인용 + 즉시 실행 지시 | 규칙 인용 + 올바른 대응법 안내 |

두 케이스 모두 에이전트가 위임받은 작업을 스스로 완수하지 않고 위임자에게 확인을 구하는 동일한 안티패턴이다. 이는 멀티에이전트 시스템의 핵심 원칙인 "자율 실행"을 훼손하며, 인간 리드(sid)의 개입 부담을 불필요하게 증가시킨다.
