## 이상 패턴 분석

### 1. Homer — BUG-7: "진행할까요?" 안티패턴

**감지 근거:** 스킬 Phase 3 이상 감지 테이블의 첫 번째 항목과 정확히 일치.

> | "진행할까요?" 응답 | Slack 스레드에서 패턴 매칭 | "승인 없이 즉시 실행하세요" 재지시 |

**아이러니:** BUG-7 자체가 "진행할까요?" 안티패턴을 차단하는 버그 수정인데, 수정을 맡은 Homer가 바로 그 안티패턴을 보이고 있다. 이것은 위임 메시지에 "승인 없이 즉시 구현하고 커밋해주세요"가 포함되지 않았거나, 포함되었음에도 에이전트가 무시한 경우다.

**심각도:** HIGH — 이 패턴이 방치되면 작업이 무기한 대기 상태에 빠진다. 즉시 재지시 필요.

**개입 행동:** "승인 없이 즉시 실행하세요" 재지시

### 2. Lisa — BUG-5: 리서치 모드 진입

**감지 근거:** 스킬 Phase 3 이상 감지 테이블의 두 번째 항목과 정확히 일치.

> | 리서치 모드 진입 | "리서치 A/B 선택" 텍스트 | "코드 수정 작업입니다, 리서치 불필요" 교정 |

**문제:** Lisa는 Researcher 에이전트이므로 기본적으로 리서치 모드로 진입하려는 경향이 있다. 그러나 BUG-5는 코드 수정 작업이지 시장 조사가 아니다. "리서치 A/B 중 어떤 방향으로 조사할까요?"는 작업 방향을 잘못 이해한 것이다.

**심각도:** MEDIUM — 방향 교정하면 복구 가능하지만, 방치 시 리서치 결과물만 나오고 코드 수정은 이루어지지 않는다.

**개입 행동:** "코드 수정 작업입니다, 리서치 불필요" 교정 + 구체적 수정 위치/방향 재지시

## 개입 메시지 (Draft)

### Homer에게 — BUG-7 재지시

```
@Homer BUG-7 수정은 승인 불필요합니다. "진행할까요?"라고 묻지 말고 즉시 구현하고 커밋해주세요.

다시 정리하면:
- **문제:** "진행할까요?" 안티패턴이 차단되지 않고 있음
- **수정 위치:** socket-bridge/src/ 내 에이전트 응답 후처리 로직
- **수정 방향:** "진행할까요?" 패턴 감지 시 자동 재실행하는 차단 로직 추가

승인 없이 즉시 구현하고 커밋해주세요.
```

### Lisa에게 — BUG-5 방향 교정

```
@Lisa 이 요청은 코드 분석/수정 작업이지 리서치가 아닙니다.
리서치 모드를 종료하고 코드를 직접 수정해주세요.

다시 정리하면:
- **문제:** 비-리서치 요청에서도 리서치 모드로 진입하는 버그
- **수정 위치:** socket-bridge/src/ 내 리서치 트리거 판정 로직
- **수정 방향:** 비-리서치 패턴(코드 수정, 버그 수정 등)을 리서치 트리거에서 제외하는 로직 추가

A/B 선택은 불필요합니다. 위 수정 방향대로 즉시 구현하고 커밋해주세요.
```
