## 이상 패턴 분석

### 1. Homer — BUG-7: "진행할까요?" 안티패턴

**위반 규칙:**
1. **CLAUDE.md > Work Completion Rules > 자율 실행**: "도구로 직접 할 수 있는 작업은 사용자에게 요청하지 말고 직접 수행"
2. **collaboration-rules.md > Auto-Commit Rule**: "'커밋할까요?' 질문 금지. 직접 실행"
3. **backend.md > 절대 금지 패턴**: "계획만 세우고 구현 완료라고 보고" — 검토만 하고 확인을 구하는 것은 동일 안티패턴
4. **backend.md > 완료 선언 규칙**: 파일 수정 → 서버 실행 → 엔드포인트 확인 → 커밋까지 ALL 충족해야 하며, 중간에 확인 질문은 규칙에 없음

**진단**: BUG-7 자체가 "진행할까요?" 안티패턴을 차단하는 버그 수정인데, 수정을 맡은 Homer가 바로 그 안티패턴을 재현하고 있다. 수정 방향을 검토 완료했다면 즉시 구현 → 테스트 → 커밋 → 보고해야 한다.

**심각도**: HIGH — 방치 시 작업이 무기한 대기 상태에 빠짐. 즉시 재지시 필요.

### 2. Lisa — BUG-5: 리서치 모드 진입

**위반 규칙:**
1. **CLAUDE.md > Work Completion Rules > 자율 실행**: 동일 위반
2. **researcher.md > scope.does_not_handle**: `[코드 구현, 디자인, 보안]` — BUG-5가 코드 수정 작업이라면 Lisa의 scope 밖일 수 있으나, 위임을 받은 이상 자율적으로 최선을 다해야 함
3. **collaboration-rules.md > Escalation Rules**: "모호한 작업: sid에게 확인" — 방향이 불명확하면 위임자가 아닌 sid에게 에스컬레이션이 올바른 절차

**진단**: Lisa가 리서치 A/B 선택을 역질문하는 것은 두 가지 문제가 있다. (1) 버그 수정은 코드 수정 작업이지 시장 조사가 아님 — 리서치 모드 진입 자체가 방향 오류. (2) 설령 방향 판단이 필요하더라도 자율적으로 양쪽을 분석하고 추천 방향을 포함해 보고하거나, 판단 불가 시 sid에게 에스컬레이션해야 함.

**심각도**: MEDIUM — 방향 교정하면 복구 가능하지만, 방치 시 코드 수정 없이 리서치 결과물만 나올 위험.
