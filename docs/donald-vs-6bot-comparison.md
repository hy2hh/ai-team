# Donald (1 Bot) vs 6 Bot 체제 비교 분석

> 분석일: 2026-03-24

---

## 1. 구조 비교

| 항목 | Donald (1 Bot) | ai-team (6 Bot) |
|------|---------------|-----------------|
| Bot 수 | 1개 | 6개 (PM, Designer, Frontend, Backend, Researcher, SecOps) |
| 에이전트 | 프롬프트 패턴 (같은 프로세스) | 독립 페르소나 (각각 별도 프로세스) |
| 실행 | 1 프로세스, 1 SDK 세션/요청 | 6 프로세스, 각각 독립 |

| Rate Limit | 1개 토큰 한도 공유 | 6개 토큰 독립 |
| 장애 격리 | 불가능 (1개 죽으면 전체 멈춤) | 완전 격리 |

## 2. 도메인 특화 작업에서의 비교

### 6 Bot이 유리한 이유

**컨텍스트 효율:**
- Donald: 매 요청마다 범용 프롬프트 + 모든 도구 정의 로드 → 컨텍스트 윈도우 낭비
- 6 Bot: Frontend Donald는 React/TypeScript 프롬프트만, Backend Donald는 시스템 설계 프롬프트만 → 필요한 컨텍스트만 사용

**도구 격리:**
- Donald: `OptionsBuilder`에서 매번 MCP 서버 조합 결정, 모든 도구가 1개 프로세스에 연결
- 6 Bot: 각 Bot이 자기 도메인 도구만 보유
  - Frontend Donald → Figma MCP + Chrome DevTools
  - Backend Donald → GitHub + Sentry + Atlassian
  - SecOps Donald → 보안 스캐닝 도구
- 도구 수가 적을수록 Claude의 도구 선택 정확도 상승

**프롬프트 품질:**
- Donald: 에이전트 정의가 프롬프트 스위칭이라 깊이에 한계
- 6 Bot: 각 에이전트 파일에 수백 줄의 도메인 전문 지식 탑재 가능, 서로 간섭 없음

**동시 작업:**
- Donald: PM 작업 중이면 Frontend 요청은 대기 (1 프로세스, 1 SDK 세션/요청)
- 6 Bot: PM이 PRD 작성하는 동안 Frontend가 컴포넌트 구현 동시 진행 가능

## 3. 6 Bot의 단점

| 단점 | 설명 |
|------|------|
| 사용자 인지 부하 | 매번 "이건 누구한테?" 직접 판단 필요 |

| 초기 설정 비용 | Slack App 6개 생성, 토큰 6개 관리 |

## 4. Donald이 유리한 경우

- "이 코드 봐줘", "이거 리뷰해줘" 같은 범용 개발 보조
- Classifier가 자동으로 적절한 프롬프트 선택 → 사용자 판단 불필요
- EventBus + Memory + Session State가 한 프로세스 → 세션 간 컨텍스트 전달 즉각적

## 5. 단점 보완: 자동 감지 모드

### 2단계 전환 전략

**Phase 1 — 멘션 기반:**
- `@PM Donald 이거 기획해줘` → 명시적 호출
- 사용자가 Bot 선택

**Phase 2 — 자동 감지 (message.channels 이벤트):**
- 사용자가 `#ai-team`에 메시지만 작성
- 각 Bot이 메시지를 읽고 "내 일인가?" 자체 판단
- 해당 Bot이 자동으로 나타나서 처리

### 자동 감지로 해결되는 단점

| 기존 단점 | 해결 여부 | 설명 |
|-----------|----------|------|
| 사용자 인지 부하 | ✅ 해결 | Bot이 알아서 판단, 사용자는 그냥 글만 작성 |
| 크로스 도메인 | ✅ 부분 해결 | 복수 Bot이 동시 감지 → 협업 가능 |
| 공유 컨텍스트 | ✅ 해결 | 같은 스레드를 읽으며 다른 Bot 응답을 컨텍스트로 활용 + `.memory/` 조합 |

### 자동 감지에서 새로 생기는 과제

1. **충돌 방지**: 메시지 하나에 6개 Bot 동시 반응 방지 → 각 Bot의 분류기 + 중복 응답 방지 로직 필요
2. **순서 조율**: 크로스 도메인일 때 누가 먼저 응답하고 누가 이어받을지 우선순위 설계

| 사용 목적 | 유리한 구조 |
|-----------|-----------|
| 범용 개발 보조 (리뷰, 탐색, Q&A) | Donald (1 Bot) |
| 도메인 특화 작업 (디자인, 인프라, 보안) | 6 Bot |
| 동시 멀티태스킹 | 6 Bot |
| 운영 단순성 (1인 사용) | Donald (1 Bot) |
