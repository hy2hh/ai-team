## 배경 & 목표

법률 서비스 접근성 문제 해결. 일반인이 법률 분쟁 상황에서 빠르게 AI 기반 판단을 받고, 필요 시 변호사와 연계할 수 있는 B2C 서비스.

| 항목 | 결정 |
|------|------|
| 타겟 | B2C 주력 |
| AI 역할 | 법률 판단 주체 (근거 제시 포함) |

---

## 핵심 기능 (MVP 범위)

### F-1. 케이스 관리
- 사용자가 분쟁 상황을 텍스트로 입력
- 케이스 저장, 조회, 진행 상태 추적
- 대시보드에서 진행 중인 사건 목록 관리

### F-2. AI 법률 판단
- 입력된 케이스에 대해 AI가 분석 수행
- 출력: 승소 가능성(%), 근거 판례, 권장 조치
- 판례 기반 RAG 검색으로 답변 신뢰도 확보

### F-3. 문서 생성
- 내용증명 자동 생성
- 계약서 검토 (위험 조항 하이라이트)
- 건별 과금 ₩9,900 적용

## 데이터 소스 전략 (확정)

| 소스 | 규모 | 용도 | 상태 |
|------|------|------|------|
| AI Hub 판례 | 9,000건 | MVP RAG 프로토타입 | 즉시 착수 |
| 법망 API (법령/대법원) | — | 법령 검색 | 무료, 즉시 연동 가능 |
| 엘박스 | 200만 건+ | 풀 커버리지 | B2B 파트너십 협상 중 |

엘박스 선정 이유: 케이스노트 인수 → 단일 파트너십으로 최대 500만 건급 커버리지

## 핵심 화면 (MVP 최소 범위)

1. *온보딩/사건 입력* — 분쟁 상황 입력 플로우
2. *AI 판단 결과* — 승소 가능성, 근거 판례, 권장 조치
3. *문서 생성* — 내용증명 생성 플로우
4. *케이스 대시보드* — 진행 중인 사건 목록

모바일 우선(Mobile-first) 설계

---

## 기술 스택 방향 (Homer 확정 예정)

- 백엔드: Node.js / Express (기존 스택 유지)
- 벡터 DB: pgvector / Chroma / Pinecone (Homer 선택)
- 임베딩: OpenAI ada-002 또는 오픈소스 한국어 모델
- 프론트: Next.js + React (기존 스택 유지)

- [ ] 모바일 우선 레이아웃

- [ ] 내용증명 템플릿 생성 및 사용자 입력값 치환

## 에러 케이스

### EC-1. RAG 검색 결과 없음
- [x] Given: 사용자 케이스와 유사한 판례가 벡터 DB에 없음
- [x] When: AI 판단 요청 실행
- [x] Then: "관련 판례를 찾지 못했습니다" 메시지 표시, AI는 일반 법률 지식 기반으로 응답하되 판례 부재 명시

### EC-2. 엘박스 API 연동 실패
- [x] Given: 엘박스 API 응답 오류 또는 타임아웃
- [x] When: 판례 검색 요청
- [x] Then: AI Hub 데이터 fallback으로 응답, 사용자에게 일부 데이터 제한 안내

### EC-3. 문서 생성 실패
- [x] Given: PDF 생성 중 오류 발생
- [x] When: 사용자가 내용증명 다운로드 요청
- [x] Then: 에러 메시지 표시, 결제 취소 또는 재시도 옵션 제공

### EC-4. AI 응답 신뢰도 경고
- [x] Given: AI 판단 신뢰도가 낮음 (판례 부족 또는 복잡한 케이스)
- [x] When: AI 판단 결과 표시
- [x] Then: "전문 변호사 상담을 권장합니다" 경고 배너 표시
