### designer
*알림 시스템 UI 컴포넌트 설계 — 토스 디자인 시스템 기반*

---

*1. 핵심 의견*

알림 시스템은 토스의 *"한 화면, 한 목적"* 원칙에 따라 3개 계층으로 분리해야 합니다: ① 실시간 피드백용 *Toast*, ② 즉각 인지용 *Badge + Dot*, ③ 전체 알림 열람용 *알림 리스트(전용 화면)*. 모달 기반 알림 센터는 사용하지 않으며, 바텀시트 또는 전용 페이지로 처리합니다.

---

*2. 근거 — 컴포넌트별 상세 스펙*

*① Toast (즉각 피드백)*

| 항목 | 스펙 |
| --- | --- |
| 위치 | 하단 탭 바 위 `12px`, 좌우 `20px` 패딩 |
| 높이 | `48px` |
| 배경 | `#191F28` (Dark surface) |
| 텍스트 | `#FFFFFF`, 16px, weight 400, letter-spacing `-0.3px` |
| Radius | `12px` |
| Shadow | `0 4px 16px rgba(0,0,0,0.12)` |
| 모션 | 하단에서 슬라이드업 spring(`1, 80, 10`), 3초 후 자동 닫힘 |
| 액션 | 선택적 우측 텍스트 버튼 (`#0064FF`, "확인하기") |
| 상태 | Default / Success(`#00C471` 아이콘) / Error(`#F04452` 아이콘) / Info(`#0064FF` 아이콘) |

```
┌─────────────────────────────────────────┐
│  ● 새 댓글이 달렸습니다          확인하기  │  ← Toast
└─────────────────────────────────────────┘
```

*② Badge & Dot (네비게이션 인지)*

| 항목 | 스펙 |
| --- | --- |
| Dot (읽지 않은 알림 존재) | `6px` 원형, `#F04452`, 아이콘 우상단 `-2px` offset |
| Count Badge | 최소 `18px` 높이, 패딩 `4px 6px`, radius `50%`, bg `#F04452`, text `#FFFFFF` 11px weight 600 |
| 숫자 표시 | 9 이하 숫자, 10 이상 `9+`, 100 이상 `99+` |
| 위치 | 하단 탭 바 알림 아이콘 + 상단 바 벨 아이콘 |

```
   🔔
  (3)  ← Count Badge on nav icon
```

*③ 알림 리스트 (전용 화면)*

페이지 구조:
```
┌──────────────────────────────────┐
│  ← 알림                    모두 읽기  │  ← 상단 바 (56px)
├──────────────────────────────────┤
│                                  │
│  ┌────────────────────────────┐  │
│  │  오늘                      │  │  ← ListHeader (13px, #8B95A1)

│  │     프로젝트 B · 1시간 전   │  │     text: #8B95A1
