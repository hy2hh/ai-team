# 회의 결정: 칸반보드 실시간 알림 기능 설계

## 배경
칸반보드에 실시간 알림 기능을 추가하려 합니다.

## 현재 코드 상태 (탐색 완료)
- *스택:* Next.js 16 + React 19, Express + SQLite, WebSocket(ws 라이브러리)
- *실시간 인프라:* WebSocket 이미 구현됨 — 카드 생성/수정/이동/삭제 이벤트를 모든 클라이언트에 브로드캐스트
- *현재 알림:* UI 토스트만 존재 (WIP 초과, 카드 이동 실패 알림)
- *브로드캐스트 이벤트:* card:created, card:updated, card:moved, card:deleted

## 논의 필요 사항
각 에이전트는 다음에 대해 독립적으로 의견을 제시하세요:

1. *구현 범위 제안:* 어떤 알림 유형을 우선 구현할지 (인앱 알림 센터, 브라우저 Push, Slack 연동 등)
2. *기술 접근 방식:* 기존 WebSocket 인프라 활용 vs 별도 알림 채널 구축
3. *데이터 모델:* 알림 저장소 필요 여부 (읽음 상태, 알림 히스토리 등)
4. *구현 순서 및 의존성:* Designer → Frontend 순서 또는 Backend → Frontend 병렬 등

*A. 알림 유형별 시각적 구분*

```
• 카드 생성:     🆕 + --color-col-todo (#4f7ef0)
• 카드 이동:     ➡️ + --color-col-inprogress (#fbbf24)
• 카드 완료:     ✅ + --color-col-done (#4ade80)
• 카드 삭제:     🗑️ + --color-priority-high (#f87171)

```

*B. 알림 센터 위치 및 구조*

• *위치:* 헤더 우측에 🔔 아이콘 + 뱃지 카운터 (읽지 않은 알림 수)
• *패널:* 클릭 시 드롭다운 또는 사이드 패널 (모바일: bottom sheet)
• *그룹핑:* 같은 카드에 대한 연속 알림은 그룹화 ("카드 X가 3번 업데이트됨")
• *시간 표시:* 상대 시간 (방금, 5분 전, 1시간 전)

*C. 토스트 vs 알림 센터 역할 분리*

• *토스트:* 즉각적 피드백 (내가 방금 한 액션의 결과 — 유지)
• *알림 센터:* 타인의 액션 기록 (히스토리 조회용 — 신규)

• notification_id: 고유 ID
• type: created | updated | moved | deleted | assigned
• card_id: 대상 카드
• actor: 액션 수행자 (에이전트명)

• is_read: boolean (읽음 여부)

• *알림 과다:* 실시간 협업 시 분당 수십 개 알림 가능 — *그룹핑 + 디바운싱* 필수
• *모바일 UX:* 현재 `@media (max-width: 479px)` 브레이크포인트에서 드롭다운 패널은 bottom sheet로 전환 필요
• *접근성:* 알림 센터는 `role="region"` + `aria-live="polite"` 적용 필요
• *스크롤 성능:* 알림 목록이 길어지면 가상 스크롤 또는 페이지네이션 고려

};
```

  metadata TEXT,                   -- JSON: { from_column, to_column, ... }
  is_read INTEGER DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now'))
);
```
