### 3. 모델 차이(sonnet vs opus)에 대한 인사이트

| 관점 | sonnet (제 경우) | opus (추정) |
| --- | --- | --- |
| 정보 합성 경향 | 검색 결과 범위 내에서만 합성 | 사전 학습 지식을 더 적극 활용 |
| 출처 미확보 시 | 해당 정보 제외 | 합리적 추론으로 포함 가능 |
| 톤 | 중립적·학술적 | 설득적·액션 지향 |
| 리스크 | 정보량 부족 가능 | hallucination 리스크 |

*핵심 가설:* opus는 더 긴 컨텍스트와 추론 능력으로 "검색에서 직접 나오지 않은 사례"도 사전 지식 기반으로 합성하는 경향이 있습니다. 이것이 *장점(풍부한 인사이트)*이자 *리스크(검증 불가 수치)*입니다.

### 4. 리스크/우려사항

• *opus 보고서의 구체적 ROI 수치 검증 필요* — `+2,622%`, `9천만 달러` 같은 수치는 매우 구체적이어서 출처가 있다면 강력한 근거가 되지만, 없다면 hallucination일 가능성도 배제 불가
• *제 보고서의 한계* — 출처 엄격주의로 인해 실무자에게 유용한 ROI 사례, 기술 최적화 팁이 누락됨
• *도구 비교 누락* — Semrush만 언급한 것은 검색 쿼리 설계의 한계 (더 구체적인 "AEO tools comparison" 쿼리 필요했음)

*🔍 QA 의견 — AIEO 리서치 보고서 품질 비교*

*1. 핵심 의견*

Lisa 보고서가 *검증 가능성(verifiability)* 측면에서 명확히 우위에 있습니다. 모든 수치에 URL 출처를 명시한 것은 리서치 품질의 핵심 요건을 충족합니다. 외부 에이전트 보고서의 구체적 수치(+2,622%, 9천만달러 등)는 출처 미명시 시 *hallucination 리스크*가 높아 신뢰도 평가가 불가합니다.

*2. 근거 (품질 기준별 평가)*

| 항목 | Lisa (sonnet) | 외부 에이전트 (opus) |
| --- | --- | --- |
| 출처 검증 가능성 | ✅ 모든 수치 URL 명시 | ❌ 일부 수치 출처 불명 |
| Hallucination 리스크 | 🟢 낮음 (검증 가능) | 🔴 높음 (검증 불가) |
| 정보 범위 | 🟡 개념/시장 중심 | ✅ 전략/기술/ROI 포함 |
| 실행 가능성 | 🟡 방향성 제시 | ✅ 구체적 액션 제시 |
| 학술적 정확성 | ✅ 보수적·중립적 | 🟡 설득적·주장 지향 |

*주요 리스크 분석:*
• `+2,622% ROI` — 출처 없이 이 수준의 구체적 수치는 LLM이 합성했을 가능성 높음
• `BrightEdge Schema 3.2배` — 검색 시 실제 BrightEdge 리포트에서 확인 필요
• `Chemours 9천만달러` — 기업 사례는 검증 가능하나 출처 링크 필수

*3. 리스크/우려사항*

• *Opus의 "합성 성향"*: Opus 모델은 더 풍부한 세부사항을 생성하지만, 출처 없는 수치는 신뢰할 수 없음. 이는 모델 특성보다 *프롬프트 설계의 문제*일 가능성이 높음 (출처 명시 강제 지시 부재)
• *Sonnet의 "보수성"*: 출처 검증 불가 정보를 제외한 것은 정확성 관점에서 올바른 판단이나, 정보 범위가 좁아지는 trade-off 발생
• *품질 vs 범위 trade-off*: 출처 있는 좁은 정보 vs 출처 없는 넓은 정보 — QA 관점에서는 전자가 우선

*4. 추천 행동*

| 우선순위 | 행동 |
| --- | --- |
| *P0* | 리서치 에이전트 프롬프트에 "모든 수치는 반드시 URL 출처 명시, 출처 없으면 '확인 필요' 표기" 규칙 추가 |
| *P1* | 외부 에이전트 보고서의 주요 수치 5개 이상 출처 역추적 검증 (WebSearch) |
| *P2* | 리서치 품질 체크리스트 수립: (1) 출처 URL 존재 (2) 출처 접근 가능 (3) 수치가 출처 내용과 일치 |
| *P3* | 모델 간 비교 실험: 동일 프롬프트로 sonnet/opus 결과 비교하여 모델 특성 vs 프롬프트 효과 분리 |

*종합 판정:*
• Lisa 보고서: *신뢰도 기준 PASS* — 출처 검증 가능
• 외부 에이전트 보고서: *신뢰도 기준 CONDITIONAL* — 출처 검증 후 재평가 필요
