## 코드 리뷰 결과

발견된 문제를 심각도 순으로 정리합니다.

---

### 🔴 Critical (버그 / 무한 루프)

**1. `useEffect` 의존성 배열 누락 → 무한 루프**

```js
// ❌ 현재: 매 렌더링마다 fetch 호출 → setUsers → 재렌더링 → 반복
useEffect(() => {
  fetch('/api/users').then(r => r.json()).then(setUsers);
});

// ✅ 수정: 마운트 시 1회만 실행
useEffect(() => {
  fetch('/api/users').then(r => r.json()).then(setUsers);
}, []);
```

**2. `key` prop 없음 → 리스트 렌더링 버그**

```jsx
// ❌ key 없음
users.map(u => <div onClick={...}>{u.name}</div>)

// ✅ 수정
users.map(u => <div key={u.id} onClick={...}>{u.name}</div>)
```

**3. 언마운트 후 `setUsers` 호출 가능 → 메모리 누수**

```js
// ✅ cleanup으로 방지
useEffect(() => {
  let cancelled = false;
  fetch('/api/users')
    .then(r => r.json())
    .then(data => { if (!cancelled) { setUsers(data); } });
  return () => { cancelled = true; };
}, []);
```

---

### 🟠 High (에러 처리 / 안정성)

**4. fetch 에러 처리 없음**

네트워크 오류나 서버 에러 시 `setUsers`에 에러 객체가 들어갈 수 있음.

```js
fetch('/api/users')
  .then(r => {
    if (!r.ok) { throw new Error('Failed to fetch'); }
    return r.json();
  })
  .then(setUsers)
  .catch(err => console.error(err)); // 또는 error state 관리
```

**5. 로딩/빈 상태 처리 없음**

초기 렌더링 시 빈 화면이 표시됨. `isLoading` 상태 추가 권장.

---

### 🟡 Medium (팀 컨벤션 위반)

**6. 인라인 화살표 함수 → 핸들러 분리 필요**

팀 컨벤션: 이벤트 핸들러는 `handleEvent` 패턴으로 분리.

```jsx
// ❌ 인라인 → 매 렌더링마다 새 함수 생성
<div onClick={() => setUsers(users.filter(x => x.id !== u.id))}>

// ✅ 분리 + useCallback
const handleRemoveUser = useCallback((id: number) => {
  setUsers(prev => prev.filter(x => x.id !== id));
}, []);

<div key={u.id} onClick={() => handleRemoveUser(u.id)}>
```

**7. `React.memo` 없음**

팀 컨벤션: 모든 컴포넌트에 `React.memo` 적용 필수.

**8. TypeScript 타입 없음**

```ts
// ✅
interface User {
  id: number;
  name: string;
}

const [users, setUsers] = useState<User[]>([]);
```

**9. TSDoc 없음**

팀 컨벤션: exported 컴포넌트에 TSDoc 필수.

### ✅ 수정된 전체 코드

```tsx
interface User {
  id: number;
  name: string;
}

/** 사용자 목록을 표시하며 클릭으로 삭제할 수 있는 컴포넌트 */
export const UserList = React.memo(() => {
  const [users, setUsers] = useState<User[]>([]);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    let cancelled = false;

    fetch('/api/users')
      .then(r => {
        if (!r.ok) { throw new Error('Failed to fetch users'); }
        return r.json();
      })
      .then((data: User[]) => {
        if (!cancelled) {
          setUsers(data);
          setIsLoading(false);
        }
      })
      .catch(err => {
        if (!cancelled) {
          setError(err.message);
          setIsLoading(false);
        }
      });

    return () => { cancelled = true; };
  }, []);

  const handleRemoveUser = useCallback((id: number) => {
    setUsers(prev => prev.filter(x => x.id !== id));
  }, []);

  if (isLoading) { return <div>로딩 중...</div>; }
  if (error) { return <div>에러: {error}</div>; }
  if (users.length === 0) { return <div>사용자가 없습니다.</div>; }

  return (
    <>
      {users.map(u => (
        <div key={u.id} onClick={() => handleRemoveUser(u.id)}>
          {u.name}
        </div>
      ))}
    </>
  );
});
```
