## 원인 분석

### 핵심 문제: 순차적 DB 쿼리 (Sequential Await)

```
현재 실행 흐름:
[query1: user]        → 500ms 대기
              [query2: orders]      → 500ms 대기
                            [query3: notif]    → 500ms 대기
                                          [query4: analytics] → 500ms 대기
총 소요: ~2000ms
```

4개의 쿼리가 서로 **독립적**임에도 불구하고 `await`를 순차적으로 사용해 직렬 실행되고 있습니다. 앞 쿼리가 끝나야 다음이 시작됩니다.

### 부가적 문제들

| 문제 | 위치 | 영향 |
|------|------|------|
| `SELECT *` 전체 컬럼 조회 | 모든 쿼리 | 불필요한 데이터 전송 |
| `orders` 결과 수 제한 없음 | orders 쿼리 | 데이터 폭증 가능 |
| 캐싱 없음 | 전체 | 반복 요청 시 매번 DB 조회 |

## 해결책

### 1️⃣ 즉각 적용 — `Promise.all()`로 병렬 실행

```js
app.get('/api/dashboard', async (req, res) => {
  const [user, orders, notifications, analytics] = await Promise.all([
    db.query('SELECT id, name, email FROM users WHERE id = ?', [req.userId]),
    db.query('SELECT id, status, total, created_at FROM orders WHERE user_id = ? ORDER BY created_at DESC LIMIT 20', [req.userId]),
    db.query('SELECT id, message, created_at FROM notifications WHERE user_id = ? AND read = false', [req.userId]),
    db.query('SELECT COUNT(*) as total FROM page_views WHERE user_id = ?', [req.userId]),
  ]);

  res.json({ user, orders, notifications, analytics });
});
```

```
개선된 실행 흐름:
[query1: user    ]
[query2: orders  ]  → 병렬 실행 → 가장 느린 쿼리 시간만큼만 대기
[query3: notif   ]
[query4: analytics]
총 소요: ~500ms (4배 단축)
```

### 2️⃣ DB 인덱스 확인

쿼리 속도를 보장하려면 아래 인덱스가 있어야 합니다:

```sql
-- 없으면 추가
CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_notifications_user_id_read ON notifications(user_id, read);
CREATE INDEX idx_page_views_user_id ON page_views(user_id);
```

실행 계획 확인:
```sql
EXPLAIN SELECT * FROM orders WHERE user_id = 123;
-- type: "ALL" → 인덱스 없음 (문제)
-- type: "ref"  → 인덱스 있음 (정상)
```

### 3️⃣ 에러 처리 + 타임아웃 추가 (운영 안정성)

```js
app.get('/api/dashboard', async (req, res) => {
  try {
    const TIMEOUT_MS = 3000;

    const [user, orders, notifications, analytics] = await Promise.race([
      Promise.all([
        db.query('SELECT id, name, email FROM users WHERE id = ?', [req.userId]),
        db.query('SELECT id, status, total, created_at FROM orders WHERE user_id = ? ORDER BY created_at DESC LIMIT 20', [req.userId]),
        db.query('SELECT id, message, created_at FROM notifications WHERE user_id = ? AND read = false', [req.userId]),
        db.query('SELECT COUNT(*) as total FROM page_views WHERE user_id = ?', [req.userId]),
      ]),
      new Promise((_, reject) =>
        setTimeout(() => reject(new Error('Query timeout')), TIMEOUT_MS)
      ),
    ]);

    res.json({ user, orders, notifications, analytics });
  } catch (err) {
    console.error('[dashboard]', err);
    res.status(500).json({ error: 'Internal server error' });
  }
});
```

## 개선 효과 요약

| 조치 | 예상 단축 |
|------|----------|
| `Promise.all()` 병렬화 | ~1,500ms |
| `SELECT *` → 필요한 컬럼만 | ~100–300ms |
| DB 인덱스 추가 | ~200–500ms |
| **합계** | **~2,000ms 이하로** |

**가장 먼저 할 것**: `Promise.all()` 적용 — 코드 한 줄 수준의 변경으로 가장 큰 효과를 얻을 수 있습니다.
