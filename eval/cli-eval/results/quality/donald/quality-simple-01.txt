## `debounce` 함수 설명

**핵심 목적**: 함수 호출을 "지연"시켜, 연속 호출 시 마지막 호출만 실행되도록 제어합니다.

---

### 동작 원리

```
호출 →  타이머 시작 (ms 후 실행 예약)
재호출 → 이전 타이머 취소 + 새 타이머 시작
재호출 → 이전 타이머 취소 + 새 타이머 시작
...
(ms 동안 조용) → 드디어 fn 실행 ✅
```

---

### 코드 흐름

| 코드 | 역할 |
|------|------|
| `let id` | 타이머 ID 보관 (클로저로 유지) |
| `clearTimeout(id)` | 이전에 예약된 실행 취소 |
| `setTimeout(() => fn(...args), ms)` | ms 후 실행 재예약 |
| `return (...args) => ...` | 원본 함수를 감싼 새 함수 반환 |

---

### 대표적인 사용 예시

```js
// 검색창 — 타이핑 중엔 API 호출 안 하고, 멈추면 호출
const handleSearch = debounce((query) => {
  fetchResults(query);
}, 300);

input.addEventListener('input', (e) => handleSearch(e.target.value));
```

사용자가 빠르게 타이핑하는 동안엔 API를 계속 호출하지 않고, **입력이 300ms 동안 없으면** 그때 한 번만 호출합니다.

---

### `throttle`과 차이점

| | debounce | throttle |
|--|----------|----------|
| 언제 실행? | 마지막 호출 후 N ms 뒤 | N ms마다 최대 1회 |
| 주 용도 | 검색, 자동저장 | 스크롤, 리사이즈 |
