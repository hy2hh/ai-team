## `debounce` 함수 설명

이 함수는 **디바운스(debounce)** 패턴을 구현한 유틸리티입니다.

---

### 핵심 개념

**"연속 호출이 끝난 뒤 일정 시간이 지나야 실제 함수를 실행한다"**

---

### 동작 순서

1. `debounce(fn, ms)`를 호출하면 **클로저(closure)** 로 감싸진 새 함수를 반환합니다.
2. 반환된 함수가 호출될 때마다:
   - `clearTimeout(id)` → **이전에 예약된 실행을 취소**
   - `setTimeout(() => fn(...args), ms)` → **새로운 실행을 `ms`ms 뒤로 예약**
3. 호출이 멈추고 `ms` 밀리초가 지나야 비로소 `fn`이 **딱 한 번** 실행됩니다.

---

### 예시로 이해하기

```js
const search = debounce((query) => {
  console.log('API 호출:', query);
}, 300);

// 사용자가 빠르게 타이핑할 때
search('r');      // ← 취소됨
search('re');     // ← 취소됨
search('rea');    // ← 취소됨
search('reac');   // ← 취소됨
search('react');  // ← 300ms 후 실행됨 ✅
```

→ 타이핑할 때마다 API를 호출하는 게 아니라, **멈춘 뒤 300ms 후에 딱 한 번만 호출**됩니다.

---

### 주로 사용하는 상황

| 상황 | 이유 |
|------|------|
| 검색창 입력 | 타이핑 중 매번 API 호출 방지 |
| 윈도우 리사이즈 이벤트 | 이벤트가 너무 자주 발생 |
| 버튼 연속 클릭 방지 | 중복 요청 방지 |

---

### 클로저 포인트

`let id`가 **반환된 함수 바깥에** 선언되어 있어서, 호출할 때마다 동일한 `id` 변수를 공유합니다. 덕분에 이전 타이머를 정확히 취소할 수 있습니다.
