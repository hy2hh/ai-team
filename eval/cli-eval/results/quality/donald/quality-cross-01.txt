## 📋 1. 기획 (Product)

### 핵심 사용자 시나리오
1. 신규 가입 → 플랜 선택 → 결제 정보 입력 → 구독 시작
2. 기존 사용자 → 플랜 업그레이드/다운그레이드
3. 구독 해지 → 만료일까지 서비스 유지
4. 결제 실패 → 재시도 알림 → 자동 재결제

### 구독 플랜 구조 (예시)
| 플랜 | 가격 | Stripe Price ID |
|------|------|-----------------|
| Free | 무료 | — |
| Pro | ₩9,900/월 | price_xxx |
| Team | ₩29,900/월 | price_yyy |

### 핵심 성공 지표
- 결제 전환율 (플랜 선택 → 결제 완료)
- 구독 유지율 (Churn Rate)
- 결제 실패율

## 🏗️ 2. 기술 아키텍처

```
[프론트엔드]                    [백엔드]                    [Stripe]
  React App
  Stripe.js  ──카드 토큰──▶  POST /subscriptions  ──API──▶  Customer
  Elements                   POST /webhooks  ◀──Webhook──   Subscription
                              GET  /billing                  Invoice
                                     │
                              [데이터베이스]
                              users.stripe_customer_id
                              subscriptions.status
                              subscriptions.current_period_end
```

### 데이터 흐름 원칙
- **카드 번호는 절대 백엔드를 거치지 않음** — Stripe.js가 직접 Stripe 서버로 전송
- 백엔드는 `PaymentMethod ID`만 수신
- 모든 구독 상태 변경은 **Webhook 이벤트 기반**으로 처리

### CheckoutForm 핵심 구현 패턴
```tsx
// CheckoutForm.tsx — Stripe Elements 활용
import { useStripe, useElements, CardElement } from '@stripe/react-stripe-js';

/** 결제 정보를 입력받고 구독을 생성하는 폼 컴포넌트 */
export const CheckoutForm = React.memo(({ planId, onSuccess }: Props) => {
  const stripe = useStripe();
  const elements = useElements();
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const handleSubmit = useCallback(async (e: React.FormEvent) => {
    e.preventDefault();
    if (!stripe || !elements) { return; }

    setIsLoading(true);
    setError(null);

    // 1. PaymentMethod 생성 (카드 정보는 Stripe로만 전송)
    const { paymentMethod, error: pmError } = await stripe.createPaymentMethod({
      type: 'card',
      card: elements.getElement(CardElement)!,
    });

    if (pmError) {
      setError(pmError.message ?? '결제 오류가 발생했습니다.');
      setIsLoading(false);
      return;
    }

    // 2. 백엔드에 PaymentMethod ID만 전송
    const result = await createSubscription({
      paymentMethodId: paymentMethod.id,
      planId,
    });

    // 3. 3DS 인증 필요 시 처리
    if (result.requiresAction) {
      const { error: confirmError } = await stripe.confirmCardPayment(
        result.clientSecret,
      );
      if (confirmError) {
        setError(confirmError.message ?? '인증 실패');
        setIsLoading(false);
        return;
      }
    }

    onSuccess();
    setIsLoading(false);
  }, [stripe, elements, planId, onSuccess]);

  return (
    <form onSubmit={handleSubmit}>
      <CardElement options={CARD_ELEMENT_OPTIONS} />
      {error && <ErrorMessage message={error} />}
      <button type="submit" disabled={!stripe || isLoading}>
        {isLoading ? '처리 중...' : '구독 시작'}
      </button>
    </form>
  );
});
```

### 상태 관리 전략
- 구독 상태: React Query(SWR) — 서버 상태
- 결제 폼 UI 상태: 로컬 useState
- 플랜 선택: URL 파라미터 (`?plan=pro`)

## 🏗️ 4. 백엔드 설계

### API 엔드포인트 명세

| Method | Path | 설명 |
|--------|------|------|
| `POST` | `/api/subscriptions` | 구독 생성 |
| `PATCH` | `/api/subscriptions/:id` | 플랜 변경 |
| `DELETE` | `/api/subscriptions/:id` | 구독 해지 |
| `GET` | `/api/billing/portal` | Stripe Customer Portal URL |
| `GET` | `/api/billing/invoices` | 청구 내역 |
| `POST` | `/api/webhooks/stripe` | Stripe Webhook 수신 |

### 구독 생성 로직
```typescript
// POST /api/subscriptions
const createSubscription = async (req: Request, res: Response) => {
  const { paymentMethodId, planId } = req.body;
  const userId = req.user.id;

  // 1. Stripe Customer 생성 또는 조회
  let customerId = await getUserStripeCustomerId(userId);
  if (!customerId) {
    const customer = await stripe.customers.create({
      email: req.user.email,
      metadata: { userId },
    });
    customerId = customer.id;
    await saveStripeCustomerId(userId, customerId);
  }

  // 2. PaymentMethod를 Customer에 연결
  await stripe.paymentMethods.attach(paymentMethodId, {
    customer: customerId,
  });

  // 3. 기본 결제 수단 설정
  await stripe.customers.update(customerId, {
    invoice_settings: { default_payment_method: paymentMethodId },
  });

  // 4. 구독 생성
  const subscription = await stripe.subscriptions.create({
    customer: customerId,
    items: [{ price: PLAN_PRICE_IDS[planId] }],
    payment_behavior: 'default_incomplete',
    expand: ['latest_invoice.payment_intent'],
  });

  // 5. 3DS 필요 여부 반환
  const paymentIntent = subscription.latest_invoice?.payment_intent;
  if (paymentIntent?.status === 'requires_action') {
    return res.json({
      requiresAction: true,
      clientSecret: paymentIntent.client_secret,
    });
  }

  return res.json({ subscriptionId: subscription.id });
};
```

### Webhook 처리 (핵심)
```typescript
// POST /api/webhooks/stripe
const handleWebhook = async (req: Request, res: Response) => {
  // 서명 검증 필수
  const sig = req.headers['stripe-signature'] as string;
  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(
      req.rawBody,  // raw body 필수 (파싱 전)
      sig,
      process.env.STRIPE_WEBHOOK_SECRET!,
    );
  } catch (err) {
    return res.status(400).json({ error: 'Webhook 서명 검증 실패' });
  }

  // 이벤트 타입별 처리
  switch (event.type) {
    case 'customer.subscription.updated':
      await syncSubscriptionStatus(event.data.object as Stripe.Subscription);
      break;
    case 'customer.subscription.deleted':
      await deactivateSubscription(event.data.object as Stripe.Subscription);
      break;
    case 'invoice.payment_succeeded':
      await handlePaymentSuccess(event.data.object as Stripe.Invoice);
      break;
    case 'invoice.payment_failed':
      await handlePaymentFailure(event.data.object as Stripe.Invoice);
      break;
  }

  res.json({ received: true });
};
```

### 데이터베이스 스키마
```sql
-- 구독 테이블
CREATE TABLE subscriptions (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id               UUID NOT NULL REFERENCES users(id),
  stripe_customer_id    VARCHAR(255) NOT NULL,
  stripe_subscription_id VARCHAR(255) UNIQUE NOT NULL,
  plan_id               VARCHAR(50) NOT NULL,
  status                VARCHAR(50) NOT NULL,  -- active, past_due, canceled, ...
  current_period_start  TIMESTAMPTZ NOT NULL,
  current_period_end    TIMESTAMPTZ NOT NULL,
  cancel_at_period_end  BOOLEAN DEFAULT FALSE,
  created_at            TIMESTAMPTZ DEFAULT NOW(),
  updated_at            TIMESTAMPTZ DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_subscriptions_user_id ON subscriptions(user_id);
CREATE INDEX idx_subscriptions_stripe_id ON subscriptions(stripe_subscription_id);
```

## 🔒 5. 보안 설계

### 카드 정보 보안
- ✅ PCI DSS 준수 — Stripe.js가 카드 정보를 직접 처리, 백엔드 무통과
- ✅ `publishableKey`만 프론트에 노출 — `secretKey`는 서버에만
- ✅ 환경변수로 키 관리 (`STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`)

### Webhook 보안
- ✅ **서명 검증 필수** — `stripe.webhooks.constructEvent()` 사용
- ✅ Raw body 파싱 미들웨어 별도 적용 (JSON 파싱 전 원본 유지)
- ✅ 멱등성(Idempotency) 처리 — 동일 이벤트 중복 처리 방지

```typescript
// 멱등성 처리 예시
const processedEvents = new Set<string>(); // 실제 환경: Redis 사용

const handleWebhook = async (event: Stripe.Event) => {
  if (processedEvents.has(event.id)) {
    return; // 중복 이벤트 무시
  }
  processedEvents.add(event.id);
  // ... 처리
};
```

### API 보안
- ✅ 모든 결제 API에 인증 미들웨어 적용
- ✅ Rate Limiting — 결제 엔드포인트 (예: 10회/분)
- ✅ 소유권 검증 — 구독 조회 시 `userId` 매칭 확인
- ✅ HTTPS 강제 — Stripe 요구사항

### 민감 데이터 로깅 금지
```typescript
// ❌ 금지
logger.info('결제 처리', { cardNumber, cvv });

// ✅ 올바른 방식
logger.info('결제 처리', { customerId, planId, subscriptionId });
```

## ⚠️ 핵심 주의사항

| 항목 | 설명 |
|------|------|
| Raw Body | Webhook 경로에 JSON 파서 적용 금지, `express.raw()` 별도 사용 |
| 구독 상태 | DB 직접 수정 금지, **반드시 Webhook 이벤트 기반**으로 동기화 |
| 플랜 변경 | `stripe.subscriptions.update()`로 처리, 일할 정산 자동 처리 |
| 테스트 카드 | `4242 4242 4242 4242` (성공), `4000 0000 0000 9995` (실패) |
| 환경 분리 | 테스트키/프로덕션키 절대 혼용 금지 |
