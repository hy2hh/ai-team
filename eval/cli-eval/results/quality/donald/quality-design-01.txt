# 사용자 인증 시스템 설계

---

## 1. 아키텍처 개요

```
Client (Browser/App)
    │
    ▼
[API Gateway / BFF]
    │
    ├── Auth Service ──────────────────────────────┐
    │       │                                      │
    │       ├── Email/Password Handler             │
    │       ├── OAuth Handler (Google, GitHub)     │
    │       └── Password Reset Handler             │
    │                                              │
    ├── Token Store (Redis)                        │
    ├── User DB (PostgreSQL)                       │
    └── Email Service (SES / SendGrid)             │
                                                   │
                        External OAuth Providers ──┘
                        (Google, GitHub)
```

## 2. 데이터 모델

### `users` 테이블
```sql
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email         VARCHAR(320) UNIQUE NOT NULL,
  email_verified BOOLEAN DEFAULT false,
  password_hash VARCHAR(255),          -- OAuth 전용은 NULL
  created_at    TIMESTAMPTZ DEFAULT now(),
  updated_at    TIMESTAMPTZ DEFAULT now(),
  locked_until  TIMESTAMPTZ,           -- 계정 잠금
  mfa_secret    VARCHAR(255),          -- TOTP 비밀키 (선택)
  mfa_enabled   BOOLEAN DEFAULT false
);

CREATE TABLE oauth_accounts (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  provider    VARCHAR(50) NOT NULL,    -- 'google' | 'github'
  provider_id VARCHAR(255) NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT now(),
  UNIQUE (provider, provider_id)
);

CREATE TABLE refresh_tokens (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  token_hash  VARCHAR(255) NOT NULL,
  expires_at  TIMESTAMPTZ NOT NULL,
  revoked     BOOLEAN DEFAULT false,
  device_info JSONB,                   -- UA, IP 등
  created_at  TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE password_reset_tokens (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  token_hash VARCHAR(255) NOT NULL,
  expires_at TIMESTAMPTZ NOT NULL,
  used       BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## 3. 기능별 설계

### 3-1. 이메일/비밀번호 로그인

**회원가입 플로우**
```
POST /auth/register
  Body: { email, password }

1. 이메일 중복 체크
2. 비밀번호 강도 검증 (규칙 → 아래 보안 섹션)
3. bcrypt(password, cost=12) → password_hash 저장
4. 이메일 인증 토큰 발송
5. 201 반환 (인증 전까지 로그인 불가)
```

**로그인 플로우**
```
POST /auth/login
  Body: { email, password }

1. 이메일로 사용자 조회 (존재 여부 timing-safe 처리)
2. 계정 잠금 여부 확인 (locked_until > now())
3. bcrypt.compare(password, hash)
4. 실패 시: login_attempts +1, 5회 이상 → 15분 잠금
5. 성공 시:
   a. Access Token (JWT, 15분) 발급
   b. Refresh Token (Opaque, 30일) 발급 → DB 저장
   c. Refresh Token은 HttpOnly Secure Cookie로 전달
6. 응답: { accessToken, user: { id, email } }
```

**토큰 갱신**
```
POST /auth/refresh
  Cookie: refreshToken (HttpOnly)

1. Cookie에서 Refresh Token 추출
2. DB에서 token_hash 조회 → revoked/만료 확인
3. 새 Access Token 발급
4. Refresh Token Rotation (구 토큰 revoke → 신 토큰 발급)
```

---

### 3-2. OAuth (Google / GitHub)

**Authorization Code Flow + PKCE**

```
[1단계] 인증 시작
GET /auth/oauth/:provider
  → code_verifier 생성, code_challenge = SHA256(code_verifier)
  → state = random(32bytes) → Redis에 5분 저장
  → 외부 OAuth 제공자로 redirect
    (scope: email, profile)

[2단계] 콜백 처리
GET /auth/oauth/:provider/callback
  Query: { code, state }

1. state 검증 (Redis 조회 → 일치 확인 후 즉시 삭제)
2. code + code_verifier → Access Token 교환 (provider API)
3. provider API로 사용자 정보 조회 (email, provider_id)
4. oauth_accounts 테이블에서 provider_id 조회
   - 존재: 해당 user_id로 로그인
   - 미존재 + 이메일 일치: 기존 계정에 OAuth 연결
   - 미존재 + 신규: users + oauth_accounts 생성
5. JWT + Refresh Token 발급 (이메일/비밀번호와 동일)
```

**Provider 설정 예시**

| 항목 | Google | GitHub |
|------|--------|--------|
| Auth URL | accounts.google.com/o/oauth2/v2/auth | github.com/login/oauth/authorize |
| Token URL | oauth2.googleapis.com/token | github.com/login/oauth/access_token |
| Scope | openid email profile | user:email |
| 이메일 미공개 처리 | — | /user/emails API 추가 조회 |

### 3-3. 비밀번호 재설정

```
[요청]
POST /auth/password-reset/request
  Body: { email }

1. 이메일 존재 여부와 무관하게 동일 응답 반환
   ("이메일이 발송되었습니다") ← 이메일 열거 방지
2. 사용자 존재 시:
   a. 기존 미사용 토큰 모두 무효화
   b. token = crypto.randomBytes(32) → hex string
   c. token_hash = SHA256(token) → DB 저장 (TTL: 1시간)
   d. 이메일 발송: https://app.com/reset?token=<raw_token>
3. Rate Limit: IP당 5회/시간

[검증 및 변경]
POST /auth/password-reset/confirm
  Body: { token, newPassword }

1. token_hash = SHA256(token) → DB 조회
2. 만료/사용 여부 확인
3. 비밀번호 강도 검증
4. bcrypt로 새 해시 저장
5. 토큰 used = true 처리
6. 해당 사용자의 모든 Refresh Token revoke (강제 재로그인)
7. 보안 알림 이메일 발송
```

## 4. 보안 요구사항

### 4-1. 비밀번호 정책
```
- 최소 12자
- 대문자, 소문자, 숫자, 특수문자 각 1개 이상
- 이메일 포함 금지
- 상위 10만개 유출 비밀번호 차단 (Have I Been Pwned API 또는 로컬 리스트)
- 해싱: bcrypt cost=12 (최소), Argon2id 권장
```

### 4-2. JWT 설계
```
Access Token:
  - Algorithm: RS256 (비대칭키)
  - TTL: 15분
  - Payload: { sub, email, iat, exp, jti }
  - jti (JWT ID): Redis 블랙리스트로 즉시 무효화 지원

Refresh Token:
  - Opaque (무작위 256-bit)
  - HttpOnly + Secure + SameSite=Strict Cookie
  - DB에 hash만 저장 (원본 저장 금지)
  - Rotation: 사용 시마다 갱신
  - 탈취 감지: 만료된 토큰으로 요청 → 해당 사용자 전체 세션 revoke
```

### 4-3. Rate Limiting

| 엔드포인트 | 제한 | 키 |
|-----------|------|-----|
| POST /auth/login | 10회/분 | IP |
| POST /auth/login | 20회/시간 | 이메일 |
| POST /auth/register | 5회/시간 | IP |
| POST /auth/password-reset/request | 5회/시간 | IP |
| GET /auth/oauth/:provider | 20회/분 | IP |

### 4-4. 추가 보안 조치
```
1. HTTPS 전용 (HSTS 헤더 포함)
2. CORS: 허용 Origin 화이트리스트
3. 모든 응답에 보안 헤더 적용
   - X-Frame-Options: DENY
   - X-Content-Type-Options: nosniff
   - Referrer-Policy: strict-origin-when-cross-origin
4. SQL Injection: Parameterized Query만 사용
5. 로그인 성공/실패, 비밀번호 변경 감사 로그 저장
6. 의심 활동 감지: 다른 지역 IP, 다수 실패 후 성공 → 알림 이메일
```

### 4-5. Timing Attack 방지
```javascript
// ❌ 잘못된 예: 사용자 미존재 시 빠른 응답 → 이메일 열거 가능
if (!user) return error();
if (!bcrypt.compare(pw, hash)) return error();

// ✅ 올바른 예: 사용자 없어도 항상 bcrypt 연산 수행
const user = await findUser(email);
const hash = user?.password_hash ?? DUMMY_HASH;
const valid = await bcrypt.compare(password, hash);
if (!user || !valid) return error(); // 동일 응답 시간
```
