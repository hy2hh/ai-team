## `debounce` 함수 설명

**핵심 개념**: 연속으로 호출되는 함수를 **마지막 호출 이후 일정 시간(ms)이 지난 뒤에만 실행**되도록 지연시킵니다.

---

### 동작 원리 (단계별)

```
fn 호출 → 타이머 초기화 → ms 후 실행
         ↑
    또 호출되면 타이머 리셋
```

1. `clearTimeout(id)` — 이전에 예약된 타이머를 **취소**
2. `setTimeout(() => fn(...args), ms)` — **새 타이머** 예약
3. ms 안에 또 호출되면 → 1번으로 돌아가 반복
4. ms 동안 **아무 호출 없을 때** → `fn` 실제 실행

---

### 실제 예시

```js
const save = debounce((text) => console.log('저장:', text), 300);

save('H');      // 타이머 시작
save('He');     // 이전 타이머 취소, 다시 시작
save('Hell');   // 이전 타이머 취소, 다시 시작
save('Hello');  // 이전 타이머 취소, 다시 시작
// 300ms 경과 → '저장: Hello' 출력 (딱 1번만)
```

---

### 주요 사용처

| 상황 | 이유 |
|------|------|
| 검색창 입력 | 타이핑 중 API 호출 방지 |
| 윈도우 resize | 이벤트가 너무 빈번하게 발생 |
| 폼 자동 저장 | 매 키입력마다 저장 방지 |
| 버튼 중복 클릭 방지 | 연타 시 마지막 클릭만 처리 |

---

### 클로저 포인트

`id` 변수가 **클로저**로 캡처되기 때문에, 반환된 함수가 호출될 때마다 같은 `id`를 공유합니다. 이 덕분에 타이머 취소/재등록이 가능합니다.
