## `debounce` 함수 설명

**핵심 목적**: 함수 호출을 "지연"시켜, 연속 호출 시 마지막 호출만 실행되도록 제어합니다.

---

### 동작 원리

```
호출 1 ─────┐
호출 2 ───┐  └─ clearTimeout (취소)
호출 3 ─┐  └─── clearTimeout (취소)
         └────── ms 후 실제 실행 ✅
```

1. 내부에 `id` (타이머 ID)를 클로저로 유지합니다.
2. 반환된 함수가 호출될 때마다 **이전 타이머를 취소** (`clearTimeout`)합니다.
3. 새 타이머를 `ms` 밀리초 후로 설정합니다.
4. `ms` 동안 추가 호출이 없으면 → 원래 함수 `fn` 이 실행됩니다.

---

### 사용 예시

```ts
// 검색 입력창 — 타이핑이 멈춘 후 300ms 뒤에만 API 호출
const handleSearch = debounce((query) => {
  fetchSearchResults(query);
}, 300);

input.addEventListener('input', (e) => handleSearch(e.target.value));
```

사용자가 "react"를 타이핑할 때:
- `r`, `re`, `rea`, `reac` 입력 → 모두 취소됨
- `react` 입력 후 300ms 동안 추가 입력 없음 → API 호출 1회 ✅

---

### 핵심 패턴 정리

| 개념 | 역할 |
|------|------|
| `let id` | 클로저로 타이머 ID 유지 |
| `clearTimeout(id)` | 연속 호출 시 이전 예약 취소 |
| `setTimeout(() => fn(...args), ms)` | ms 후 실행 예약 |
| `...args` | 원래 함수의 인자를 그대로 전달 |

> **활용 사례**: 검색 입력, 윈도우 리사이즈 핸들러, 자동 저장 기능 등 **빈번한 이벤트를 제어**할 때 자주 사용합니다.
