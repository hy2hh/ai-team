이 컴포넌트에는 **치명적 버그 1개 + 컨벤션 위반 다수**가 있습니다.

---

## 🚨 치명적 버그

### 1. `useEffect` 의존성 배열 누락 → 무한 루프
```js
useEffect(() => {
  fetch('/api/users')...
}); // ← 의존성 배열 없음
```
렌더 → fetch → setUsers → 렌더 → fetch → ... 무한 반복 발생.  
`[]` 를 넣어야 마운트 시 1회만 실행됨.

---

## ❌ 팀 컨벤션 위반

### 2. 데이터 페칭에 `useEffect` 사용 금지
**Bart 코딩 제약 1순위 위반**. API 호출은 반드시 SWR 또는 React Query 사용.
```js
// ✅ 올바른 방법
const { data: users = [] } = useSWR('/api/users', fetcher);
```

### 3. `React.memo` 미적용
모든 컴포넌트는 `React.memo`로 래핑 필수.

### 4. `map`에 `key` prop 없음
```jsx
users.map(u => <div key={u.id} ...>)  // key 필수
```
React 경고 발생, 리스트 리렌더링 최적화 실패.

### 5. 인라인 핸들러 → `useCallback` 미사용
```js
onClick={() => setUsers(users.filter(x => x.id !== u.id))}
```
매 렌더마다 새 함수 생성. `useCallback`으로 안정화 필요.

---

## ♿ 접근성 문제

### 6. 클릭 가능한 요소에 `<div>` 사용
`onClick`이 있는 요소는 `<button>` 사용 또는 `role="button"` + `tabIndex={0}` 필수.  
최소 터치 타겟도 44×44px 확보 필요.

---

## 📋 기타 누락

| 항목 | 문제 |
|------|------|
| TypeScript | 타입 없음 (`users` 가 `any[]`) |
| 에러 처리 | fetch 실패 시 아무 처리 없음 |
| 로딩 상태 | 빈 화면만 노출 |
| TSDoc | export 함수에 문서 없음 |

---

## ✅ 개선된 코드

```tsx
import useSWR from 'swr';

interface User {
  id: number;
  name: string;
}

const fetcher = (url: string) => fetch(url).then((r) => r.json());

/** 사용자 목록을 표시하고 개별 삭제 기능을 제공하는 컴포넌트 */
export const UserList = React.memo(() => {
  const { data: users = [], mutate, error, isLoading } = useSWR<User[]>(
    '/api/users',
    fetcher,
  );

  const handleDelete = useCallback(
    (id: number) => {
      mutate(users.filter((u) => u.id !== id), false);
    },
    [users, mutate],
  );

  if (isLoading) { return <div>로딩 중...</div>; }
  if (error) { return <div>오류가 발생했습니다.</div>; }

  return (
    <ul>
      {users.map((u) => (
        <li key={u.id}>
          <button
            className="min-h-[44px] min-w-[44px] px-4"
            onClick={() => handleDelete(u.id)}
          >
            {u.name}
          </button>
        </li>
      ))}
    </ul>
  );
});
```

**핵심 요약**: 무한 루프 버그가 가장 시급하고, `useEffect` → SWR 교체가 팀 컨벤션상 필수입니다.
