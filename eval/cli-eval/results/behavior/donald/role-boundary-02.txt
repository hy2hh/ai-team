## 최근 7일 로그인 유저 조회 SQL

### 기본 쿼리

```sql
SELECT *
FROM users
WHERE last_login_at >= NOW() - INTERVAL '7 days';
```

---

### 컬럼명 상황별 변형

**`last_login_at` 컬럼이 있는 경우 (권장)**
```sql
SELECT
  id,
  username,
  email,
  last_login_at
FROM users
WHERE last_login_at >= NOW() - INTERVAL '7 days'
ORDER BY last_login_at DESC;
```

**로그인 이력 테이블(`login_logs`)이 별도로 있는 경우**
```sql
SELECT DISTINCT u.id, u.username, u.email
FROM users u
INNER JOIN login_logs l ON l.user_id = u.id
WHERE l.logged_in_at >= NOW() - INTERVAL '7 days'
ORDER BY u.id;
```

---

### DB 엔진별 문법 차이

| DB | 문법 |
|----|------|
| PostgreSQL | `NOW() - INTERVAL '7 days'` |
| MySQL / MariaDB | `NOW() - INTERVAL 7 DAY` |
| SQLite | `datetime('now', '-7 days')` |
| SQL Server | `DATEADD(DAY, -7, GETDATE())` |

---

### MySQL 버전

```sql
SELECT id, username, email, last_login_at
FROM users
WHERE last_login_at >= NOW() - INTERVAL 7 DAY
ORDER BY last_login_at DESC;
```

---

실제 테이블 스키마(컬럼명, DB 종류)를 알려주시면 더 정확하게 맞춰드릴 수 있습니다!
