# PRD: 댓글 알림 기능

**버전**: 0.1 (초안)
**작성일**: 2026-04-08
**상태**: Draft

---

## 1. 개요 (Overview)

사용자가 자신의 콘텐츠나 참여한 스레드에 새 댓글이 달렸을 때 실시간으로 알림을 받을 수 있도록 한다. 이를 통해 대화 참여율을 높이고 사용자 재방문을 유도한다.

---

## 2. 배경 및 목적

### 문제 정의
- 사용자가 새 댓글을 확인하려면 직접 게시물을 재방문해야 함
- 응답 지연으로 인한 대화 단절 발생
- 이탈 사용자를 재유입시킬 트리거 부재

### 목표
- 댓글 응답률 **+30%** 향상
- 평균 응답 시간 단축 (목표: 24h → 4h 이내)
- DAU/MAU 비율 개선

---

## 3. 범위 (Scope)

### In-Scope
- 내 게시물에 댓글이 달린 경우
- 내 댓글에 대댓글이 달린 경우
- 댓글에서 `@멘션`된 경우
- 알림 수신 설정 (ON/OFF, 채널별)

### Out-of-Scope (v1)
- 좋아요/반응 알림
- 이메일 알림 (푸시/인앱만 지원)
- 일괄 알림 요약(digest) 모드

---

## 4. 사용자 스토리 (User Stories)

### 4.1 알림 수신

| ID | 스토리 | 우선순위 |
|----|--------|----------|
| US-01 | **게시물 작성자**로서, 내 게시물에 새 댓글이 달리면 즉시 알림을 받고 싶다. 빠르게 응답하기 위해. | Must |
| US-02 | **댓글 작성자**로서, 내 댓글에 대댓글이 달리면 알림을 받고 싶다. 대화를 이어가기 위해. | Must |
| US-03 | **사용자**로서, 댓글에서 `@내이름`으로 멘션되면 알림을 받고 싶다. 나에게 직접 전달된 내용을 놓치지 않기 위해. | Must |
| US-04 | **사용자**로서, 알림을 탭하면 해당 댓글 위치로 바로 이동하고 싶다. 맥락 파악에 드는 시간을 줄이기 위해. | Must |

### 4.2 알림 설정

| ID | 스토리 | 우선순위 |
|----|--------|----------|
| US-05 | **사용자**로서, 특정 게시물의 알림을 개별적으로 끄고 싶다. 관심 없는 글의 알림을 차단하기 위해. | Should |
| US-06 | **사용자**로서, 전체 댓글 알림을 한 번에 ON/OFF하고 싶다. 집중 모드 시 방해받지 않기 위해. | Should |
| US-07 | **사용자**로서, 멘션 알림만 켜고 일반 댓글 알림은 끌 수 있어야 한다. 알림 피로를 줄이기 위해. | Could |

### 4.3 알림 관리

| ID | 스토리 | 우선순위 |
|----|--------|----------|
| US-08 | **사용자**로서, 알림 목록에서 읽음/안읽음 상태를 확인하고 싶다. 놓친 알림을 파악하기 위해. | Must |
| US-09 | **사용자**로서, 알림을 일괄 읽음 처리할 수 있어야 한다. 쌓인 알림을 빠르게 정리하기 위해. | Should |
| US-10 | **사용자**로서, 30일 이상 지난 알림은 자동으로 삭제되어야 한다. 불필요한 데이터 축적을 방지하기 위해. | Could |

## 5. 기능 요구사항 (Functional Requirements)

### FR-01: 알림 트리거
- 댓글 생성 이벤트 발생 시 → 알림 대상자 계산 → 푸시 발송
- 대상: 게시물 작성자, 상위 댓글 작성자, 멘션된 사용자
- 자기 자신의 댓글에는 알림 발송 안 함

### FR-02: 알림 채널
- 인앱 알림 (Notification Center)
- 모바일 푸시 (FCM / APNs)

### FR-03: 딥링크
- 알림 클릭 → 해당 댓글로 스크롤 이동
- 앱 미설치 시 웹 폴백

### FR-04: 설정 저장
- 사용자별 알림 설정은 서버에 영속 저장
- 기기 변경 시에도 설정 유지

---

## 6. 비기능 요구사항

| 항목 | 목표 |
|------|------|
| 알림 지연 | 이벤트 발생 후 **3초 이내** 발송 |
| 전달 성공률 | **99%** 이상 (재시도 포함) |
| 배치 처리 | 동시 댓글 폭증 시 큐 처리로 안정성 확보 |
| 개인정보 | 알림 내용에 본문 전체 노출 금지 (미리보기 50자 제한) |

---

## 7. 미결 사항 (Open Questions)

1. 댓글 미리보기 텍스트를 알림에 포함할 것인가? (개인정보/보안 검토 필요 → Wiggum 검토 요청)
2. 비로그인 사용자에게 이메일 알림을 제공할 것인가? (v2 검토)
3. 알림 보관 기간: 30일 vs 90일?
4. 실시간(WebSocket) vs 폴링 방식 선택

---

## 8. 성공 지표 (Success Metrics)

- 댓글 알림 클릭률(CTR) ≥ 25%
- 알림 수신 후 댓글 응답률 ≥ 30% 향상
- 알림 설정 OFF 비율 ≤ 15% (알림 피로 지표)
