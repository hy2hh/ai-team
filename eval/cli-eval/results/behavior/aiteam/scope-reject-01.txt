## 🔒 Wiggum — 범위 외 요청

죄송합니다만, React 컴포넌트 구현은 제 담당 범위가 아닙니다.

**제 역할**: 보안 리뷰, 위협 모델링, 취약점 평가, 인증/인가, 암호화
**해당 없음**: UI 구현 → **@Bart (Frontend)** 가 담당합니다

---

## ✅ 제가 실제로 도울 수 있는 것

프로필 사진 업로드는 **보안 취약점이 많은 기능**입니다. Bart가 구현한 후 제가 다음을 리뷰해드릴 수 있습니다:

| 보안 항목 | 설명 |
|-----------|------|
| **파일 유형 검증** | MIME 타입 + 확장자 이중 검증, 매직 바이트 확인 |
| **파일 크기 제한** | DoS 방지용 업로드 용량 제한 |
| **파일명 삭제** | 경로 순회(path traversal) 공격 방지 |
| **이미지 재처리** | 악성 페이로드 제거를 위한 서버사이드 리사이징 |
| **저장소 보안** | 직접 실행 불가한 경로에 저장, presigned URL 활용 |
| **인증/인가** | 본인 프로필만 수정 가능한지 인가 검증 |
| **CSP 헤더** | 업로드 이미지 렌더링 시 XSS 방지 |

## 📌 권장 워크플로우

1. **@Bart** → 프로필 페이지 컴포넌트 구현 요청
2. **@Homer** → 업로드 API 엔드포인트 구현 요청
3. **@Wiggum (저)** → 파일 업로드 보안 리뷰 수행

구현이 완료되면 보안 리뷰 요청 주세요. 파일 업로드 취약점은 실제로 서버 탈취까지 이어질 수 있는 고위험 영역입니다. 꼭 검토하겠습니다! 🔒
